## Übung (Directory Permissions und Octal Notation)

### Angabe
Oktale Umwandlung: Gib die chmod-Befehle mit Oktalwerten für folgende Zielzustände an:
rwxr-xr-x
rw-rw----
rwx------
Pub-Verzeichnis: Erstelle das Verzeichnis /home/kali/pub. Setze mit einem Befehl in oktaler Notation die Rechte so: Der Besitzer darf alles (7), die Gruppe darf den Inhalt auflisten und in das Verzeichnis wechseln (5), alle anderen dürfen nur in das Verzeichnis wechseln, aber den Inhalt nicht auflisten (1).
Recursive Lockdown: Du möchtest das Verzeichnis /tmp/lab/drafts/ und alle darin enthaltenen Dateien rekursiv für die Gruppe music lesbar (r) und beschreibbar (w) machen, während others keinen Zugriff haben. Welcher Befehl ist dafür nötig?
Search Permission: Welchen Befehl musst du auf dein Home-Verzeichnis /home/kali anwenden, damit andere User zwar keine Dateien darin auflisten (ls) können, aber in dein bereits existierendes Unterverzeichnis /home/kali/pub wechseln (cd) dürfen?
Analyse: Du führst den Befehl chmod -R 660 /tmp/lab/ aus. Warum können die User danach nicht mehr mit cd in die Unterverzeichnisse wechseln? Welcher Permission-Typ fehlt den Directories nun?

### Lösung
```
rwxr-xr-x  ->755
rw-rw----  ->660
rwx------  ->700
```
* Verzeichnis anlegen: sudo mkdir -p /home/kali/pub
bei mir funktionierte es nicht ohne die richtigen Berechtigungen 

* Permission setzen: sudo chmod 751 /home/kali/pub

* Recursive Lockdown Berechtigungen setzten: chmod -R g+rwX,o-rwx /tmp/lab/drafts/

* Search Permission: chmod o=x /home/kali

* Den Verzeichnissen fehlt Execute-Recht -> daher ist kein cd mehr möglich.

## Übung (Webserver Directory Permissions II)

### Angabe
Erstelle unter Kali Linux in /var/www ein Verzeichnis data und sorge dafür, dass dieses Verzeichnis nur User der Gruppe www-data verwenden können. Ein Lesezugriff auf den Directoryinhalt durch alle anderen User soll möglich sein jedoch kein Lesen der Dateiinhalte. User der Gruppe www-data können Datei-Inhalte anderer User ansehen aber nicht verändern.

Erweiterung: User der Gruppe www-data sollen die Dateien anderer User der gleichen Gruppe im Directory data nicht löschen bzw. umbenennen können.
Erweiterung: Alle Dateien/Verzeichnisse die im Ordner data erzeugt werden sollen automatisch der Gruppe www-data zugeordnet werden.
Teste mittels verschiedener User!

### Lösung

* Verzeichnis erstellen: sudo mkdir /var/www/data

* Gruppe setzen: sudo chown root:www-data /var/www/data

* Grundrechte setzen:
   - Gruppe www-data darf alles im Verzeichnis nutzen
   - Others dürfen nur auflisten (kein Dateizugriff)
* sudo chmod 2755 /var/www/data
