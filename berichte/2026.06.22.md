## Übung 1 – GNU Debugger: Passwortabfrage umgehen

```gdb
(gdb) b main
Note: breakpoint 1 also set at pc 0x555555557172.
Breakpoint 2 at 0x555555557172: file ./pwcheck.cpp, line 27.
(gdb) run

Breakpoint 1, main () at ./pwcheck.cpp:27
warning: 27	./pwcheck.cpp: No such file or directory
(gdb) disas
```

```asm
0x00005555555571a2 <+66>:    cmp    $0x0,%eax
0x00005555555571a5 <+69>:    je     0x55555555723d <main()+221>
```

Das `je` springt zum "Passwort falsch"-Zweig, wenn die Passwortprüfung 0 zurückgibt.       
Um das zu umgehen, ersetze ich diesen Befehl einfach mit `nop`-Opcodes (0x90).          
Ein `je` ist 6 Byte lang, also brauche ich 6x `nop`:          

```gdb
(gdb) set {unsigned char}0x00005555555551a5 = 0x90
(gdb) set {unsigned char}0x00005555555551a6 = 0x90
(gdb) set {unsigned char}0x00005555555551a7 = 0x90
(gdb) set {unsigned char}0x00005555555551a8 = 0x90
(gdb) set {unsigned char}0x00005555555551a9 = 0x90
(gdb) set {unsigned char}0x00005555555551aa = 0x90
```

Damit wird der Jump einfach ignoriert und das Programm läuft so weiter, als wäre das Passwort korrekt gewesen.
